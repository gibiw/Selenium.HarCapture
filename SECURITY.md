# Security Policy

## Supported Versions

Only the latest release line receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.3.x   | Yes                |
| < 0.3   | No                 |

## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

To report a vulnerability, email the maintainer directly:

**dmit.gridnev@gmail.com**

Please include:
- A description of the vulnerability and its potential impact
- Steps to reproduce or a minimal proof-of-concept
- The affected version(s)

**Expected response time:** within 72 hours of receipt.

## Disclosure Policy

This project follows **coordinated disclosure**. Once a fix is available and a patched release is published, the details of the vulnerability will be disclosed publicly. We ask that you refrain from public disclosure until the fix is released.
